## BACKEND: FastAPI (api/)

### api/main.py
Full FastAPI app with CORS enabled:

POST /api/simulate
- Body: { event: str, marketFocus: list[str], personaMix: dict, swarmSize: int }
- Creates simulation job, returns { simulationId: str }

GET /api/simulation/{id}/status
- Returns: { status: "running"|"complete", progress: int, total: int, agentResults: list, sentiment: dict, topSignals: list }

GET /api/simulation/{id}/results
- Returns full results: { demandChange: float, demandCurve: list, geoData: dict, behavioral: dict, narrative: str, attestation: dict }

GET /api/demo/pre-mortem
- Returns pre-populated Red Sea scenario results (hardcoded demo data)

### Simulation Engine (api/simulate.py)
Mock engine that:
1. Creates N "agent personas" based on persona mix
2. For each agent (processed in batches of 8 using asyncio):
   - If OPENAI_API_KEY is set, calls GPT-4o-mini with a persona prompt
   - Otherwise uses realistic mock data
   - Each returns: sentiment (-1 to 1), behavioral response text, key factors
3. K-On aggregation: average sentiments weighted by persona mix -> demand signal
4. Generate demand curve: 30/60/90 day projections with confidence bands
5. Generate geo distribution
6. Generate K-On narrative synthesis

Agent Persona Prompts (when using real LLM):
- Retail Consumer: "You are a retail consumer in {region}. Based on this news event: {event}. How does this change your purchasing behavior? Respond with JSON: {sentiment: float -1 to 1, response: str, factors: list}"
- B2B Buyer: "You are a B2B procurement manager. Event: {event}. How does this affect your sourcing decisions? Respond with JSON: {sentiment: float, response: str, factors: list}"
- Institutional Trader: "You are an institutional trader. Event: {event}. Provide behavioral response with JSON: {sentiment: float, response: str, factors: list}"

Use in-memory dict to store simulation state (simulations = {}).
Use asyncio background tasks in FastAPI.
